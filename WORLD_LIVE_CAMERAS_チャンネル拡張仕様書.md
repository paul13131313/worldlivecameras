# WORLD LIVE CAMERAS — チャンネル拡張仕様書

> 世界中の「今」を、テーマ別チャンネルで切り替えて眺めるプラットフォーム

---

## コンセプト

現在の WORLD LIVE CAMERAS は「EARTH」チャンネル（世界の風景・野生動物）のみ。
これを**テレビのチャンネルのように切り替えられるマルチチャンネルプラットフォーム**に拡張する。

ユーザーは画面上部のチャンネルバーで切り替えるだけで、
「世界の今のニュース」「世界の今の音楽」「世界の今の海」が12画面で一覧できる。

## UI/UX設計

### チャンネルバー（画面上部）

```
[ 🌍 EARTH ] [ 📺 NEWS ] [ 🎵 MUSIC ] [ 🌊 OCEAN ] [ 🏙️ CITY ]
[ 🚂 RAIL ]  [ 🌋 VOLCANO ] [ 🌌 SPACE ] [ 🐾 ANIMAL ] [ ⛈️ WEATHER ]
```

- 横スクロール（モバイル）/ 2段表示（PC）
- 現在のチャンネルはハイライト表示（白背景 + 黒文字）
- チャンネル切り替え時のトランジション: フェードイン（0.3s）
- キーボード: ← → で前後のチャンネルに切り替え
- 数字キー 1〜0 でチャンネル直接選択

### 全画面モード（既存のFキー対応を維持）

- Fキーで全画面表示（ヘッダー・チャンネルバー非表示）
- 全画面中でも ← → キーでチャンネル切り替え可能
- 全画面中は画面下部に半透明のチャンネル名を表示（3秒後にフェードアウト）

### 音声制御

- デフォルト: 全ミュート
- カードをクリック: その映像だけ音ON（他はミュート）
- MUSICチャンネルでは特に重要: 1つだけ音を出す
- Mキー: 全ミュート切り替え

### レスポンシブ

- PC: 4列×3段（12枠）
- タブレット: 3列×4段
- モバイル: 2列×6段（コンパクト表示）

## 技術設計

### データ構造

```typescript
interface Camera {
  name: string;
  channelId: string;        // YouTube チャンネルID
  videoId?: string;         // 固定VIDEO_IDの場合
  country: string;
  flag: string;
  timezone: string;
  category: string;
}

interface Channel {
  id: string;               // 'earth', 'news', 'music', etc.
  name: string;
  icon: string;
  slots: Slot[];
}

interface Slot {
  cameras: Camera[];        // メイン + 予備カメラ
  activeCameraIndex: number;
  status: 'live' | 'stream' | 'offline';
}
```

### チャンネル切り替えロジック

1. チャンネル切り替え時、現在のiframeを全て破棄
2. 新チャンネルの12スロットを生成
3. 各スロットの1番目のカメラから接続開始
4. OFFLINE検知 → 次の予備カメラに切り替え
5. 前のチャンネルの接続状態はメモリにキャッシュ
   → 戻った時に高速再表示（最後に生きていたカメラから開始）

### パフォーマンス

- 画面外（非表示チャンネル）のiframeは破棄してメモリ節約
- YouTube iframe APIのplayVideo/pauseVideoで制御
- Intersection Observer で画面外のスロットは読み込み遅延
